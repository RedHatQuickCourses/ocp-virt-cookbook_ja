Webコンソールを使用したテンプレートからのVM作成

概要

このチュートリアルでは、OpenShift Webコンソールと事前構築済みテンプレートを使用して仮想マシン（VM）を作成する方法を説明します。Webコンソールは、VM管理のための直感的なグラフィカルインターフェースを提供するため、新規ユーザーにとって理想的な出発点となります。

前提条件

  • OpenShift VirtualizationオペレーターがインストールされたOpenShift 4.18以降（OpenShift Virtualizationのインストールを参照）

  • VMを作成する権限を持つOpenShift Webコンソールへのアクセス

  • VMディスクプロビジョニング用に構成されたデフォルトのStorageClass

  • クラスター内で利用可能なブートソースイメージ（DataSource）

ブートソースが利用可能であることを確認します。

oc get datasources -n openshift-virtualization-os-images

ステップ 1: 仮想化セクションへのアクセス

  1. OpenShift Webコンソールにログインします。

  2. 左側のナビゲーションメニューで、Virtualization をクリックします。

  3. VirtualMachines をクリックして、VM管理インターフェースを表示します。

VirtualMachinesページには、選択したプロジェクト内のすべてのVMと、そのステータスおよび基本情報が表示されます。

ステップ 2: 利用可能なテンプレートの参照

OpenShift Virtualizationには、一般的なオペレーティングシステム用の事前構成済みテンプレートが含まれています。

  1. VirtualMachinesページから、Create → From template をクリックします。

  2. オペレーティングシステムごとに整理された利用可能なテンプレートを参照します。

    • Red Hat Enterprise Linux - さまざまなRHELバージョン

    • Fedora - 最新のFedoraリリース

    • CentOS Stream - CentOS Streamバージョン

    • Microsoft Windows - Windows Serverおよびデスクトップバージョン

Source available と表示されているテンプレートには、すぐに使用できる事前構成済みのブートソースがあります。これには、クラスター内で デフォルトのStorageClass が構成されている必要があります。

各テンプレートには以下が含まれています。

  • 事前構成済みのCPUおよびメモリ設定

  • OSに適したディスクサイズ

  • ブートソース参照（DataSource）

  • Cloud-initまたはsysprep構成（該当する場合）

ステップ 3: テンプレートからのVM作成

この例では、Fedora VMを作成します。他のオペレーティングシステムの場合も手順は同様です。

テンプレートの選択

  1. Fedora テンプレートタイルをクリックします。

  2. サイドパネルに表示されるテンプレートの詳細を確認します。

  3. Customize VirtualMachine をクリックします。

VM詳細の構成

Customize VirtualMachine ページで、Overview タブのVM設定を構成します。

  1. Name: VMの名前を入力します（例: fedora-test-vm）。

  2. Project: VMを作成するプロジェクト（ネームスペース）を選択します。

  3. Boot source: デフォルトでFedoraのDataSourceが表示されているはずです。

  4. CPU: CPUコア数を調整します（通常、デフォルトは1）。

  5. Memory: RAM割り当てを調整します（デフォルト値はテンプレートによって異なります）。

ストレージのカスタマイズ（オプション）

Disks タブをクリックしてストレージを変更します。

  1. ルートディスクの構成を確認します。

  2. 必要に応じてディスクサイズを調整したり、追加のディスクを追加したりします。

  3. StorageClassがニーズに適していることを確認します。

確認と作成

  1. タブ全体の設定を確認します。

  2. Create VirtualMachine をクリックします。

  3. 必要に応じて、Start this VirtualMachine after creation にチェックを入れて、作成後にすぐに起動します。

ステップ 4: VM作成の監視

作成をクリックすると、VM詳細ページにリダイレクトされます。

プロビジョニングの進捗確認

VMは作成中にいくつかのフェーズを通過します。

  1. Provisioning - ブートソースからディスクがクローンされています。

  2. Starting - VMが起動しています（起動が選択されている場合）。

  3. Running - VMが完全に動作しています。

Events タブを監視して、詳細なプロビジョニング手順を確認します。

# Alternatively, watch events from CLI
oc get events -n <your-namespace> --field-selector involvedObject.name=<vm-name> -w

VMが実行中であることの確認

プロビジョニングが完了すると、ステータスが Running に変わります。

ステップ 5: VMコンソールへのアクセス

Webコンソールは、VMのグラフィカルコンソールへの直接アクセスを提供します。

コンソールの起動

  1. VM詳細ページで、Console タブをクリックします。

  2. ブラウザ内で直接VNCコンソールが読み込まれます。

コンソールのオプション

コンソールツールバーでは、以下のオプションが提供されます。

  • Send key - 特殊なキーの組み合わせを送信（Ctrl+Alt+Delなど）

  • Disconnect - コンソール接続を閉じる

  • Guest credentials - テンプレートが提供するユーザー名とパスワード

パフォーマンスを向上させるには、日常的な作業ではWebコンソールではなく、Linux VMへのSSHアクセス または Windows VMへのRDP を使用することを検討してください。

ゲストへのログイン

Fedoraやその他のクラウドイメージの場合、デフォルトの資格情報はcloud-init構成に依存します。

  • cloud-initユーザーデータが提供されていない場合は、ユーザーのセットアップが必要になることがあります。

  • デフォルトの資格情報については、テンプレートのドキュメントを確認してください。

  • 自動化されたユーザーセットアップを行うために、テンプレート内のcloud-initをカスタマイズすることを検討してください。

ステップ 6: 基本的なVM操作

Webコンソールは、一般的なVM操作への容易なアクセスを提供します。

VMアクションメニュー

VM詳細ページまたはVirtualMachinesリストから、アクションメニューにアクセスします。

VMの停止

  1. Actions → Stop をクリックします。

  2. メッセージが表示されたら、アクションを確定します。

  3. VMが正常にシャットダウンされます。

# CLI equivalent
virtctl stop <vm-name>

VMの再起動

  1. Actions → Restart をクリックします。

  2. VMが正常に再起動します。

# CLI equivalent
virtctl restart <vm-name>

一時停止と一時停止の解除

VMを一時停止すると、シャットダウンせずにその状態が凍結されます。

  1. Actions → Pause をクリックします。

  2. VMの状態がメモリ内に保持されます。

  3. Actions → Unpause をクリックして再開します。

一時停止中のVMは、ノード上のメモリリソースを引き続き消費します。

VMの削除

  1. Actions → Delete をクリックします。

  2. 関連付けられたディスク（PVC）を削除するかどうかを選択します。

  3. 削除を確定します。

削除後もVMディスク（DataVolume）を保持するには、確認ダイアログに表示される Delete disk オプションのチェックを外します。これにより、後で新しいVMでそのディスクを再利用できます。
ディスクを含めてVMを削除すると、元に戻すことはできません。必要に応じてバックアップがあることを確認してください。

ステップ 7: VMの詳細とメトリクスの表示

VM詳細ページでは、監視および構成に関する情報を提供します。

Overviewタブ

一目で主要な情報を確認できます。

  • VMのステータスと状態

  • IPアドレス（ゲストエージェントがインストールされている場合）

  • CPUおよびメモリの使用率

  • ブート順序とデバイス

Metricsタブ

リアルタイムのパフォーマンスメトリクスを表示します。

  • 時間経過に伴うCPU使用率

  • メモリ使用率

  • ネットワークスループット

  • ストレージI/O

完全なメトリクスを表示するには、VM内にQEMUゲストエージェントがインストールされている必要があります。

YAMLタブ

VM定義を直接表示および編集します。

  1. YAML タブをクリックします。

  2. VirtualMachineマニフェストを確認または変更します。

  3. Save をクリックして変更を適用します。

Download ボタンをクリックして、YAMLファイルをダウンロードすることもできます。

クイック作成を使用したVMの作成

デフォルト設定を使用してより迅速にVMを作成するには、クイック作成オプションを使用します。

  1. VirtualMachinesページから、Create → From template をクリックします。

  2. テンプレートを選択します。

  3. Customize VirtualMachine ではなく、Quick create VirtualMachine をクリックします。

  4. VMがデフォルト設定ですぐに作成されます。

これは以下の場合に便利です。

  • テスト環境や開発環境

  • デフォルトのテンプレート設定で問題ない場合

  • ラピッドプロトタイピング
